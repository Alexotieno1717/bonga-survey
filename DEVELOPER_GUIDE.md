# Bonga Survey — Developer Guide

This guide documents the architecture, conventions, and day-to-day workflows for contributors.

## Architecture Overview
- Laravel 12 backend with streamlined bootstrap in `bootstrap/app.php`.
- Inertia v2 + React 19 SPA in `resources/js` with optional SSR entry `resources/js/ssr.tsx`.
- Tailwind CSS v4 with custom design tokens in `resources/css/app.css`.
- Auth via Laravel Fortify; API/SPA tokens via Sanctum.
- Typed route helpers generated by Wayfinder, with Ziggy available for raw URL generation.

### Domains
- Surveys: `Survey`, `Question`, `Option`, `SurveyResponse`, `SurveyResponseAnswer`, `SurveyMessage`.
- Phonebook: `Contact`, `ContactGroup`, `ContactGroupMap`.
- Settings: Profile, Password, Two-Factor, Appearance.

### Notable Relationships
- `Survey` has many `Question`, `SurveyResponse`, `SurveyMessage`; belongs to creator `User`; belongsToMany `Contact` via `contact_survey` (pivot holds SMS flow columns).
- `Question` belongs to `Survey`, has many `Option`, has many `SurveyResponseAnswer`.
- `Contact` belongs to `User` and optional `ContactGroup`; belongsToMany `ContactGroup` via maps; belongsToMany `Survey`; has many `SurveyResponse`, `SurveyMessage`.

## Repository Layout (selected)
- `app/` — Laravel app code (Controllers, Models, Actions, Requests)
- `bootstrap/` — App bootstrap and middleware/routing configuration
- `config/` — Laravel configs
- `database/` — Migrations, factories, seeders
- `resources/js/` — Frontend SPA (pages, components, layouts, hooks, routes/actions)
- `resources/css/app.css` — Tailwind v4 entry + theme variables
- `routes/web.php`, `routes/settings.php` — HTTP routes
- `tests/` — Pest test suite

## Backend Conventions
- PHP 8+ types everywhere; explicit return types.
- Form Requests for validation (see `app/Http/Requests/**`).
- Eloquent relationships with proper return types (`HasMany`, `BelongsTo`, etc.).
- Avoid `DB::` unless necessary; prefer query builder and Eloquent.
- Middleware configured in `bootstrap/app.php` (Laravel 12 pattern), not `Http/Kernel.php`.

### Controllers
- `SurveyController` drives survey CRUD, question management, recipients, publishing, cancel/reactivate, and responses.
- `SurveyAiController` provides AI-assisted survey drafting and application.
- `SurveySmsWebhookController` ingests inbound SMS from providers.
- Phonebook controllers manage CRUD for contacts, groups, and maps.

### Actions & Requests
- Actions encapsulate domain logic (e.g., `App/Actions/Survey/CreateSurvey`, `HandleIncomingSurveyMessage`).
- Form Requests: `StoreSurveyRequest`, `UpdateSurveyDetailsRequest`, `UpdateSurveyQuestionRequest`, etc.

## Frontend Conventions
- Pages live in `resources/js/pages/**` and are resolved by Inertia using Vite glob imports.
- Components in `resources/js/components/**`; follow existing composition patterns.
- Styling: Tailwind utility-first with tokens from CSS variables; dark mode via `.dark` class.
- Routing from TS: use Wayfinder-generated functions from `resources/js/routes/**` or `resources/js/actions/**`.
- Forms: use Inertia `useForm` or Formik/Yup where appropriate; Sonner for toasts.

### Inertia Entrypoints
- Client: `resources/js/app.tsx`
- SSR: `resources/js/ssr.tsx`

## Routing
- See ROUTES.md for a summary.
- Named routes are used with `route()` in PHP and Wayfinder/Ziggy in TS.

## Local Development
```bash
composer install && npm ci
cp .env.example .env && php artisan key:generate
php artisan migrate --force

# Start dev stack (PHP server + queues + Vite)
composer run dev

# With SSR rendering and pail logs
composer run dev:ssr
```

## Testing
- Test runner: Pest v4.
- Run: `php artisan test --compact`.
- Filter: `php artisan test --compact --filter=YourTestName`.
- Use model factories; prefer feature tests. Do not delete tests without approval.

## Quality & Style
- PHP: Laravel Pint (`composer format` runs Rector then Pint). Run Pint before committing.
- JS/TS: ESLint (flat config) + Prettier + Tailwind plugin.
- Types: strict TypeScript; `tsconfig.json` sets `@/*` alias to `resources/js/*`.

## Debugging & DX
- Laravel Pail for local logs; Laravel Boost tools available.
- Browser notifications via Sonner; check browser and server logs when debugging.
- If you don’t see UI changes, run `npm run dev` or `npm run build`.

## Common Tasks
### Add a new page
1. Create `resources/js/pages/your/Path.tsx`.
2. Add a route in `routes/*.php` returning `Inertia::render('your/Path')`.
3. Use Wayfinder route helper in components: `import yourRoute from '@/routes/your/route';`.

### Add a backend route and use it in TS
1. Define route with a name via `->name('feature.action')`.
2. Use generated Wayfinder function: `import { feature } from '@/routes'; feature.action()` or in `@/actions` for controllers.

### Handle inbound SMS provider
1. Point provider webhook to `/sms/incoming` or `/sms/webhook`.
2. Ensure payload includes recognizable phone and message keys (controller normalizes common variants).

## Deployment Notes
- Build assets: `npm run build && npm run build:ssr` if SSR is enabled.
- Configure APP_URL, APP_ENV, database, queues, and cache.
- Configure your reverse proxy to forward requests and SSL.
